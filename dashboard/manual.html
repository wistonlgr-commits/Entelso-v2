Field Operations Guide

This document outlines the standard operating procedures for utilizing the Entelso WhatsApp Integration for inventory management. The system provides real-time access to the database for field personnel, with all operations executed via direct messaging.

Security & Authentication (PIN Authorization)
A 6-digit Security PIN is strictly required for any operation that modifies asset status, location, or condition. This PIN serves as a digital signature and permanently logs the transaction against the assigned technician profile.

Supported Operations

1. Asset Query

Retrieve the current location, status, and assignment details of any registered asset.

  • Procedure: Send a query containing the asset ID (e.g., "Locate asset 27367" or "Where is DW-20V-459?").
  • Required Data: Asset ID.

2. Asset Reassignment (Checkout/Transfer)

Log the transfer of an asset from the warehouse to a specific zone, or between zones.

  • Procedure: Specify the asset, the destination zone, and provide authorization. (e.g., "Transferring asset 27367 to NSW zone, PIN 123456").
  • Required Data: Asset ID, Destination Zone, 6-digit PIN.

3. Maintenance & Damage Reporting

Log a fault report to immediately flag an asset for maintenance and prevent further deployment.

  • Procedure: Describe the fault and authorize the report. (e.g., "Asset DW-20V has a damaged power cord. PIN 123456").
  • Required Data: Asset ID, Fault Description, 6-digit PIN.

4. Asset Ingestion

Register newly acquired equipment into the active inventory database.

  • Procedure: State the intent to register, providing the new ID and description. (e.g., "Register new asset. ID: 88899, Description: Cordless Drill").
  • Required Data: Asset ID, Description.

5. Photographic Documentation

Upload visual evidence of an asset's condition upon checkout or return. This procedure is mandatory for high-value equipment.

  • Procedure: Send the photograph through WhatsApp. Include the Asset ID and PIN in the image caption. (e.g., "Evidence for 27367, PIN 123456").
  • Required Data: Image attachment, Asset ID, 6-digit PIN.
  • Batch Uploads: When submitting multiple photographs sequentially, authorization is only required on the initial image. Subsequent images will be automatically associated with the active session.

System Guidelines

  • Input Processing: The integration processes natural language. Rigid command structures are not required.
  • Media Support: The system currently supports text and image payloads exclusively. Voice memos and document attachments are not processed.
  • QR Integration: Scanning a physical asset's QR code will automatically initiate a pre-formatted query protocol in WhatsApp.

Administrative Dashboard Guide

The Entelso Web Dashboard provides centralized control and visibility over the entire asset lifecycle, enabling real-time tracking, auditing, and bulk management for administrative personnel.

1. Interface & Analytics

The primary interface provides a consolidated view of inventory metrics and status distributions.

  • Global Metrics: Real-time calculation of asset availability and maintenance ratios.
  • Data Filtering: Dynamic table filtering by Status, Operational Zone, and Assigned Team.
  • Search Indexing: Instant querying against asset serial numbers and descriptions.

2. Asset Profile Management

Selecting any record from the primary table opens the Asset Details Panel, providing a comprehensive audit trail and management interface.

Details Panel Components

  • QR Generation: Export and print unique QR identifiers for physical tagging and tracking.
  • Media Gallery: Review photographic evidence submitted by field technicians.
  • Audit Trail (Activity History): An immutable ledger logging all status changes, location transfers, and the respective authorizing personnel.

Record Modification

Administrative users possess override capabilities for all asset parameters. Access the modification interface via the Edit icon.

  • Calibration Tracking: Update 'Last Tested' and 'Next Test Due' timestamps.
  • Manual Reassignment: Force-update asset locations or assignments independently of field reports.
  • Media Management: Upload new documentation or deprecate obsolete images.
Data Persistence
Modifications made within the details panel remain local to the client session until explicitly committed to the database via the Save Changes protocol.

3. Bulk Operations

For high-volume inventory management, the system supports simultaneous multi-record updates.

  1. Select the target assets using the primary table checkboxes (☑).
  2. Access the Bulk Actions control bar that appears above the grid.
  3. Execute localized updates for Category, Zone, Status, or Team across all selected entities simultaneously.

4. Manual Asset Creation

While field technicians can perform basic ingestion, administrators can initialize complete asset profiles with extended parameters.

  1. Initiate the Add Asset protocol from the primary navigation header.
  2. Populate the comprehensive technical profile, including scheduling parameters and initial deployment status.
  3. Commit the record to generate the unique database identifier.

5. System Telemetry

The dashboard receives synchronous updates from the WhatsApp integration. All field reports, transfers, and media uploads executed by technicians are reflected instantaneously in the active table and corresponding audit trails, requiring no manual reconciliation.
